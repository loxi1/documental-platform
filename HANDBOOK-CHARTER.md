# Engineering Handbook Charter

## Propósito

El Engineering Handbook existe para preservar el conocimiento técnico, funcional y operativo de Documental Platform.

El objetivo es que ninguna decisión importante dependa exclusivamente de conversaciones, reuniones, correos o memoria personal.

## Principios

1. El repositorio es la fuente oficial.
2. El Handbook es parte del producto.
3. Toda decisión importante requiere evidencia documental.
4. Toda regla de negocio debe tener Source of Truth.
5. El backend gobierna el dominio.
6. El frontend representa el dominio.
7. Toda funcionalidad relevante actualiza código, pruebas y documentación.

## Alcance

Este Charter aplica a:

- arquitectura;
- dominio;
- backend;
- frontend;
- infraestructura;
- seguridad;
- operación;
- despliegue;
- QA;
- documentación.

## Regla final

Si una decisión no está en el repositorio, no es oficial.
