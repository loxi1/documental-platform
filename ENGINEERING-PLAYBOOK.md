# Engineering Playbook

## Flujo oficial

```text
Entender negocio
↓
Modelar dominio
↓
Registrar decisión ADR
↓
Definir API
↓
Documentar Business Flow
↓
Actualizar Data Dictionary
↓
Implementar
↓
Probar
↓
Actualizar Handbook
↓
Release
```

## Principios

1. Primero negocio, luego código.
2. Primero dominio, luego base de datos.
3. Backend gobierna el dominio.
4. Frontend representa el dominio.
5. Todo cambio importante actualiza el Handbook.
